# 종의교회 홈페이지 — 운영 안내서

홈페이지를 수정·운영하는 방법을 정리한 문서입니다. 코드를 몰라도 메모장(또는 VS Code)으로 파일을 열어 글자만 바꾸면 됩니다.

## 파일 구성

```
종의교회 홈페이지/
├── index.html        ← 홈 (히어로 · 종의정신 · 예배 라이브 · 교회 소식 · 교회 안내)
├── about.html        ← 종의소개 (설립 · 종의정신 · 사람들 · 조직)
├── worship.html      ← 종의예배 (모임 안내 · 예배 순서 · 금요묵상 · 절기)
├── life.html         ← 종의생활 (성도 생활 · 교인 등록 · 헌금 · 목회방침)
├── news.html         ← 종의소식 (주보 아카이브 · 유튜브 · 밴드)
├── css/style.css     ← 디자인 (색 · 글꼴 · 여백)
├── js/main.js        ← 메뉴 · 스크롤 효과
├── bulletins/2026/   ← 주보 PDF를 넣는 폴더
└── assets/           ← 로고 · 파비콘
```

## 1. 매주 주보 올리기 (가장 자주 하는 일)

1. Keynote에서 주보를 연 뒤 **파일 → 내보내기 → PDF**로 저장합니다.
   (주보 원본 위치: iCloud `종의교회/[2] 예배부/[2-1] 주일예배/1. 주일_주보/`)
2. PDF 파일명을 날짜로 바꿔 `bulletins/2026/` 폴더에 넣습니다 → 예: `2026-06-14.pdf`
3. `news.html`을 열어 `▼ 새 주보 추가 방법` 주석 안의 `<li> ... </li>` 한 덩어리를 복사해
   목록 **맨 위**에 붙여넣고, 날짜 두 군데(`href`의 파일명, `2026. 06. 14` 표기)를 바꿉니다.
4. 같은 파일 위쪽 `이번 주 주보 (PDF)` 버튼의 `href` 파일명도 새 날짜로 바꿉니다.

해가 바뀌면: `bulletins/2027/` 폴더를 만들고, `news.html`에 `<h2 class="year">2027</h2>`와 새 `<ul>` 목록을 2026 위에 추가합니다.

## 2. 로고 (적용 완료)

공식 로고가 이미 적용되어 있습니다. 원본은 iCloud `종의교회/[0] 자료/2. 화면/로고/`에 있고, 사이트에는 아래 파일로 들어가 있습니다.

- `assets/logo.png` — 전체 로고 (홈 화면 상단)
- `assets/logo-mark.png` — 마크만 (모든 페이지 헤더)
- `assets/favicon.png`, `assets/apple-touch-icon.png` — 브라우저 아이콘

로고가 바뀌면 위 파일들을 같은 이름으로 교체하면 됩니다. (현재 파일들은 흰 배경에서 쓸 수 있게 배경을 투명 처리한 버전입니다.)
사이트 색상(화이트 베이스, 블루 `#428CB9`, 옐로 `#F4C430`)은 `css/style.css` 맨 위 변수에서 관리합니다.

## 3. 2026년 7월 주일예배 시간 변경 시

주일예배가 오전 10시로 바뀌면 아래를 수정합니다.

1. `worship.html` 모임 안내: `오전 11:00` → `오전 10:00`, 바로 아래 "※ 7월부터 변경 예정" 문장 삭제
2. **모든 html 파일(5개)의 푸터**: `주일예배 오전 11:00` → `오전 10:00`
3. `index.html`, `worship.html` 맨 위 `<meta name="description" ...>` 안의 시간
4. `index.html` 오시는 길: "주일 오전 11시, 예배당에서 뵙겠습니다."

## 4-0. 예배 라이브 (홈 '예배 라이브' 패널)

홈의 파란 라이브 패널은 `https://youtube.com/@jongs_community/live`로 연결됩니다. 이 주소는 **실시간 방송 중이면 자동으로 라이브 영상으로, 아닐 때는 채널로** 이동하므로 매주 손댈 필요가 없습니다.
- 송출 시간이 바뀌면 `index.html`에서 `LIVE · 주일 오전 11:00` 문구만 수정하세요.
- 채널 주소가 바뀌면 `youtube.com/@jongs_community` 가 들어간 부분(헤더·이 패널·푸터)을 함께 바꿔주세요.

## 4-1. 공지 팝업 (홈 화면에 뜨는 알림창)

홈(`index.html`) 맨 아래 `<!-- ===== 공지 팝업 ===== -->` 블록에서 관리합니다.

- **켜기 / 끄기**: `data-active="true"` → `"false"`로 바꾸면 팝업이 안 뜹니다.
- **내용 바꾸기**: `promo-title`(제목), `promo-text`(본문), 버튼 글자/링크(`promo-cta`의 `href`)를 수정합니다.
  - 제목에서 강조하고 싶은 부분은 `<em>...</em>`로 감싸면 파란 그라데이션이 됩니다.
  - 본문에서 굵게 강조는 `<strong>...</strong>`.
- **새 공지로 교체할 때는 꼭 `data-key` 값도 새 이름으로 바꾸세요** (예: `2026-여름-예배시간` → `2026-가을-수련회`).
  방문자가 '오늘 하루 보지 않기'를 눌렀어도, key가 바뀌면 새 공지는 다시 보입니다.
- **이미지 넣기(선택)**: 카드 안 `<!-- <figure class="promo-figure">... -->` 주석을 풀고 `assets/`에 넣은 이미지로 `src`를 바꾸면 카드 상단에 사진이 들어갑니다.
- 방문자는 X·'닫기'·바깥 영역·Esc로 닫을 수 있고, '오늘 하루 보지 않기'를 누르면 그날 하루는 다시 뜨지 않습니다.

### 수정 사항을 사이트에 반영하는 방법

방법 A — 터미널 (이 폴더에서):
```
git add -A
git commit -m "수정 내용"
git push
```

방법 B — 웹: 저장소 페이지에서 **Add file → Upload files**로 수정한 파일을 끌어다 놓고 **Commit changes**

푸시 후 1–2분 안에 사이트에 자동 반영됩니다.
디자인(css)을 수정했다면 모든 html의 `style.css?v=3`에서 숫자를 하나 올려주세요(`?v=4`) — 방문자 브라우저 캐시 때문입니다.

도메인을 구입하면: 저장소 **Settings → Pages → Custom domain**에 입력하면 됩니다.
