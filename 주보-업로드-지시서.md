# 종의교회 주보 일괄 업로드 지시서 (Claude Code 작업용)

> 이 문서는 **로컬 Claude Code 세션이 그대로 읽고 바로 작업**하도록 쓰였습니다.
> 사용자가 "주보 올려줘 / 이 지시서대로 해줘"라고 하면 아래 절차를 순서대로 수행하세요.

---

## 0. 한 줄 요약

원본 폴더의 주보 파일들을 `bulletins/<연도>/<YYYY-MM-DD>.확장자` 로 정리하고, 목록(`bulletins.json`)을 다시 만든 뒤 `main`에 커밋·푸시한다. 핵심 작업은 **명령 하나**로 끝난다:

```bash
node scripts/import-bulletins.mjs "<원본_폴더_경로>"
```

---

## 1. 배경 (이 저장소가 주보를 다루는 방식)

- 이 저장소는 종의교회 홈페이지이며 **GitHub Pages가 `main` 브랜치에서 배포**한다. (도메인: https://www.jongschurch.com)
- 주보 파일은 `bulletins/<연도>/<YYYY-MM-DD>.<확장자>` 에 보관한다.
  - 예: `bulletins/2023/2023-02-05.pdf`
- 지원 형식: **PDF, JPG, JPEG, PNG, WEBP**
- `scripts/build-bulletins.mjs` 가 `bulletins/` 를 훑어 **`bulletins.json`** 을 생성한다.
- `news.html`(종의소식 페이지)이 `bulletins.json` 을 읽어 **연도별·최신순으로 목록을 자동 렌더링**한다.
  - 상단 "이번 주 주보" 버튼도 **가장 최신 주보로 자동 연결**된다.
- 즉, **파일만 올바른 위치에 두고 `bulletins.json` 만 갱신하면** 사이트에 자동 반영된다. HTML을 손댈 필요 없다.

---

## 2. 시작 전 확인

1. 현재 위치가 이 저장소 루트인지 확인한다. (`bulletins/`, `scripts/build-bulletins.mjs`, `news.html` 이 보여야 함)
2. 최신화한다:
   ```bash
   git checkout main
   git pull origin main
   ```
3. **원본 폴더 경로**를 확보한다. 사용자가 안 알려줬으면 **반드시 물어본다.**
   - 예: `~/Desktop/주보모음`, `/Users/이름/Downloads/2024주보`

---

## 3. 작업 절차

### 3-1. 임포트 (자동)

```bash
node scripts/import-bulletins.mjs "<원본_폴더_경로>"
```

이 스크립트가 자동으로 하는 일:
- 원본 폴더(하위 폴더 포함)의 모든 주보 파일을 찾는다.
- 파일명에서 **날짜(YYYYMMDD 또는 YYYY-MM-DD)** 를 추출한다.
  - 한글·기타 텍스트가 섞여 있어도 됨. 예) `20230205_종의교회_주보(06).pdf` → `2023-02-05`
- `bulletins/<연도>/<YYYY-MM-DD>.<확장자>` 로 **복사**한다.
- **같은 날짜에 여러 장**이면 `-2`, `-3` … 을 붙인다. (예: `2023-02-05.pdf`, `2023-02-05-2.pdf`)
- 이미 같은 내용이 있으면 **건너뛴다** (덮어쓰지 않음, 여러 번 실행해도 안전).
- 끝나면 `bulletins.json` 을 다시 만들고 **요약**을 출력한다.

### 3-2. 출력 검토 (중요)

스크립트 요약에서 다음을 **반드시 사용자에게 보고**한다:
- 복사한 주보 개수 / 연도별 개수
- **"날짜를 못 읽어 건너뛴 파일"** 목록 → 이 파일들은 자동 처리 안 됨.
  - 파일명에 날짜가 없거나 형식이 다른 경우다. 사용자에게 날짜를 확인받아
    `bulletins/<연도>/<YYYY-MM-DD>.확장자` 로 **수동 복사**한 뒤 `node scripts/build-bulletins.mjs` 를 다시 실행한다.

### 3-3. 검증

```bash
node -e "const j=require('./bulletins.json'); const by={}; j.items.forEach(i=>by[i.year]=(by[i.year]||0)+1); console.log('총', j.count, '개', by)"
```
- 연도별 개수가 기대와 맞는지 확인한다.
- (선택) 로컬 미리보기:
  ```bash
  python3 -m http.server 8099   # 다른 터미널에서
  # 브라우저로 http://localhost:8099/news.html 열어 목록 확인
  ```

### 3-4. 커밋 & 푸시

```bash
git add -A
git commit -m "주보 업로드 — <연도/범위> N개"
git push origin main
```
- GitHub Pages가 `main` 에서 배포하므로 푸시 후 1~2분이면 사이트에 반영된다.

---

## 4. 파일명 / 날짜 규칙 (요약)

| 상황 | 처리 |
|---|---|
| `2024-03-10.pdf` | 그대로 `bulletins/2024/2024-03-10.pdf` |
| `20240310_주보.pdf` (한글 포함) | 날짜만 인식 → `bulletins/2024/2024-03-10.pdf` |
| 같은 날짜 여러 장 | `2024-03-10.pdf`, `2024-03-10-2.pdf` … |
| 이미지(JPG/PNG/WEBP) | PDF와 동일하게 처리, 목록엔 "이미지" 배지 |
| 파일명에 날짜 없음 | **건너뜀** → 사용자에게 보고 후 수동 처리 |

---

## 4-2. 재정 보고도 함께 기록 (중요)

주보 안에는 거의 항상 **[3] 재정 보고** 표가 있다 (헌금 수입: 일반·목적·구제). 주보를 올릴 때 이 숫자도 같이 `finance.json` 에 기록한다.

### 어떻게
1. 각 주보(특히 **이미지로 올라온 안쪽면**, 또는 PDF 2페이지)를 **Read 도구로 열어 [3] 재정 보고 표를 읽는다.**
2. 그 주보에 적힌 **완료된 주간 행**(숫자가 채워진 줄)만 골라낸다. 비어 있는(미래) 주는 무시한다.
3. 읽은 숫자를 **헬퍼 스크립트로 입력**한다 (JSON 직접 수정 금지 — 형식이 깨질 수 있다):
   ```bash
   node scripts/set-finance.mjs <YYYY-MM-DD> <일반> <목적> <구제> ["메모"]
   # 예) node scripts/set-finance.mjs 2025-06-08 460042 100000 0 "목적: 태국 지원"
   ```
   - `<YYYY-MM-DD>`: 그 헌금 주간의 **시작 일요일** (표의 "06/08-06/14" 이면 `2025-06-08`)
   - 금액은 콤마가 있어도 됨(`460,042` OK). 이미 그 주가 있으면 **금액만 갱신**, 없으면 새로 추가.
   - `메모`: 목적헌금 용도가 있으면 (예: `"목적: 태국 지원"`). 없으면 생략.
   - `finance.json` 골격에는 **2023-01-01 ~ 최근까지 모든 주가 0원으로 이미 들어 있다.** 백필은 그 0을 실제 숫자로 바꾸는 작업이다.

### 주의
- **헌금 수입만** 기록한다 (지출/잔액은 넣지 않는다 — 공개 범위 결정사항).
- 금액은 **주보에 적힌 그대로 정확히** 옮긴다. 숫자가 흐릿하면 추측하지 말고 사용자에게 확인.
- 재정 표를 못 읽었거나 애매하면 건너뛰고 **사용자에게 보고**한다.
- `finance.json` 을 손으로 고쳤다면 `node -e "require('./finance.json')"` 로 문법 검증.

> 재정 내역은 종의소식 페이지의 **[재정 보고]** 영역(`#finance`)에 연도·월별로 자동 집계되어 표시된다.

---

## 5. 주의사항

- **`main` 에 직접 커밋·푸시**한다. (Pages가 main 에서 배포)
- 단일 파일 **100MB 초과 금지** (GitHub 제한). 큰 스캔본은 PDF로 압축 권장.
- 여러 장짜리 주보는 **PDF 한 파일**로 묶는 게 목록이 가장 깔끔하다 (한 줄로 표시).
- 기존 파일을 **덮어쓰거나 삭제하지 않는다.** 의심되면 사용자에게 먼저 확인.
- `bulletins.json` 은 **직접 손으로 고치지 말 것.** 항상 `node scripts/build-bulletins.mjs` 로 생성.
