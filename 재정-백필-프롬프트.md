# 재정(헌금) 데이터 백필 프롬프트 — 로컬 Claude Code용

종의소식 페이지의 **재정 보고**는 `finance.json` 을 읽어 자동 표시됩니다.
이미 **2023-01-01 ~ 최근까지 모든 주가 0원으로** 들어 있고, 백필은 그 0을 실제 숫자로 바꾸는 일입니다.
입력은 항상 **헬퍼 스크립트**로 합니다 (JSON 직접 수정 금지):

```bash
node scripts/set-finance.mjs <YYYY-MM-DD> <일반> <목적> <구제> ["메모"]
```

---

## 프롬프트 A — 내가 숫자를 직접 줄 때 (가장 빠름)

아래를 복사한 뒤, 표/목록을 붙여넣어 쓰세요.

```
종의교회 저장소에서 재정(헌금 수입) 데이터를 입력해줘.
입력은 반드시 scripts/set-finance.mjs 헬퍼로 하고, finance.json은 직접 수정하지 마.

규칙:
- 날짜는 그 헌금 주간의 시작 일요일(YYYY-MM-DD)
- 금액은 일반/목적/구제 순서, 콤마 있어도 됨
- 목적헌금 용도가 있으면 메모로 ("목적: 태국 지원" 등)

끝나면: node -e "require('./finance.json')" 로 문법 확인 →
git add finance.json && git commit -m "재정 데이터 입력" && git push origin main

아래가 데이터야:
2025-06-01  일반 610,000  목적 0        구제 0
2025-06-08  일반 460,042  목적 100,000  구제 0  (태국 지원)
...
```

## 프롬프트 B — 주보 PDF에서 자동으로 읽어 채울 때

```
종의교회 저장소에서 주보를 읽어 재정(헌금 수입)을 백필해줘.
먼저 `주보-업로드-지시서.md` 의 "4-2. 재정 보고도 함께 기록" 을 읽고 그대로 진행해.

진행:
1) git pull origin main
2) bulletins/<연도>/ 의 주보를 월별로 열어 [3] 재정 보고 표를 읽어.
   - 한 달치 숫자는 보통 그 달 마지막 주(또는 다음 달 첫 주) 주보에 다 들어 있어.
   - 숫자가 채워진 '완료된 주'만 입력하고, 빈(미래) 주는 건너뛰어.
3) 각 주를 scripts/set-finance.mjs 로 입력 (일반/목적/구제, 목적 용도는 메모).
4) 한 연도 끝낼 때마다: node -e "require('./finance.json')" 확인 →
   git add finance.json && git commit -m "재정 백필 — <연도>" && git push origin main
5) 못 읽었거나 애매한 주는 목록으로 보고해.

주의:
- 헌금 수입만 (지출/잔액 제외). 금액은 주보 그대로 정확히.
- PDF가 안 읽히면 알려줘 (poppler 설치 필요할 수 있음: brew install poppler).
2023년부터 순서대로 해줘.
```
